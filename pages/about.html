Source-of-Truth Rule

GitHub is canonical.

GitHub remains the source of truth for static site files, markdown assets, JSON data, and repo structure. Other tools support the workflow, but GitHub is the place that determines what the site actually serves.

  • Static HTML, CSS, JavaScript, and JSON live in the repository.
  • Markdown assets should be stored in predictable repo folders.
  • Vercel deploys from the repo.
  • Exported assets can live in Google Drive.
  • Status can be tracked in Notion.

Avoid

Do not let random Drive files, Notion notes, or chat outputs become the source of truth. That is how folder goblins seize power.

Adding Future Partners

Clone the structure, not the chaos.

Future Partners should be added with a clear profile, Partner lane, referral audiences, recommended assets, safe positioning, and a launch kit page.

  1. Create the Partner profile. Define name, lane, audience, and summary.
  2. Map referral lanes. Identify who already knows business owners with relevant funding problems.
  3. Attach assets. Link scripts, trackers, handouts, prompts, and compliance resources.
  4. Write safe positioning. Use exploratory language and provider criteria language.
  5. Add the launch kit. Update JSON and create the static page when ready.
